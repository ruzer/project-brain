# Support

## Where to ask what

- Bugs: open a bug report in GitHub Issues.
- Feature ideas: open a feature request in GitHub Issues.
- Usage questions and design discussion: use GitHub Discussions once enabled.
- Security problems: follow [SECURITY.md](SECURITY.md) instead of opening a public issue.

## What to include

- exact command
- target repository shape
- output path
- generated artifact or report that looks wrong
- current commit or release version

## What maintainers will usually ask for

- minimal reproduction
- expected behavior
- actual behavior
- relevant logs or report excerpts
